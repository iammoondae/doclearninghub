This avoids repetition (for example, "teacher uploads materials" should live in the Teacher Module, while supported file types belong in the Learning Materials Module).

Some implementation notes:

* **DOCX exam specification** is an excellent idea. Instead of requiring teachers to enter questions one by one, define a standard exam template with metadata (title, timing, answer key, points, etc.) that the system imports automatically.
* **Chemistry support** (equations, Lewis structures, reaction schemes, periodic table) is definitely feasible using libraries such as MathJax/KaTeX for equations and ChemDoodle, Kekule.js, or Ketcher for molecular structures.
* **Lockdown browser** in a web browser cannot be made completely secure. You can implement practical protections (fullscreen mode, tab-switch detection, copy/paste blocking, print blocking, context menu disabling, visibility change detection), but determined users can still circumvent them. The system should warn instructors that these measures reduce cheating but do not guarantee prevention.

Another improvement would be to classify every feature by development priority:

* **Core (MVP)** – required for the first working version.
* **Enhanced** – planned after MVP.
* **Future** – implemented in the interface but disabled (greyed out).

For example:

| Module                | MVP | Enhanced | Future |
| --------------------- | --- | -------- | ------ |
| Learning Materials    | ✓   |          |        |
| Quiz Import (DOCX)    | ✓   |          |        |
| Exam Scheduling       | ✓   |          |        |
| Randomized Questions  | ✓   |          |        |
| Question Banks        |     | ✓        |        |
| Learning Analytics    |     | ✓        |        |
| Student Notes         |     |          | ✓      |

This makes implementation much more manageable.

# DoC Learning Hub LMS/VLE System

A lightweight Learning Management System (LMS) and Virtual Learning Environment (VLE) designed primarily for university courses. The system focuses on secure content distribution, online assessments, class management, and basic academic record management while remaining significantly simpler than enterprise LMS platforms.

# 10. Chemistry Stockroom Clearance & Requisition (Fully Implemented)

## Laboratory Account Role & Dashboard
* Stockroom Dashboard for managing clearance requirements and user accountabilities.
* Excel accountability sheet upload and client-side parsing (via SheetJS).
* Intelligent student identity matching (linking roster names with active emails, holding unlinked students).
* Search and filter interface for student/teacher accountabilities by semester.
* Configurable "Semester End Date" with automated two-week reminders.

## Laboratory Requisition Form (Student Portal)
* "Lab Requisition" tab enabled for students enrolled in laboratory classes.
* Pre-populated group members' family names based on assigned classroom groupings.
* Advanced datepicker validation requiring submissions >= 3 days in advance of conduct.
* Double-column interface: (a) Chemicals/Reagents, (b) Materials/Glasswares.
* Dedicated "Unknown Samples & Codes" field.
* Attempt history list with single-click edit & resubmit for returned requisitions.

## Requisition Review Inbox (Stockroom Portal)
* Review inbox filtering requests by status (Pending, Approved, Returned).
* Complete itemized double-column preview and unknowns text viewer.
* Actions to Approve (with optional remarks) or Return for Revision (requires remarks).

## Subject Requests & Classroom Grouping Warnings
* "Subject Type" selection (Lecture vs. Laboratory) during teacher class requests.
* Auto-checking for unassigned student groups in laboratory classes with dismissible teacher warnings.

## Targeted Alerts
* Warning cards on Student Home Dashboard for outstanding accountabilities.
* Warning cards on Teacher Home Dashboard listing students with pending requirements.
